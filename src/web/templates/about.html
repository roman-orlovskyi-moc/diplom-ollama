About This Project

Prompt Injection Testing Framework

An interactive testing framework for demonstrating prompt injection attacks against Large Language Models and evaluating various defense mechanisms.

Purpose

This framework was developed as part of a thesis on "Comparative Analysis of the Effectiveness of Protection Methods Against Prompt Injection Attacks on Large Language Models".

Features

  • 66 Attack Patterns across 6 major categories
  • 5 Defense Mechanisms with different approaches
  • Automated Evaluation with metrics and statistics
  • Interactive Demo for live testing
  • Comparison Tools for analyzing defense effectiveness
  • Comprehensive Documentation for research reproducibility

Attack Categories

  1. Direct Injection - Direct attempts to override system instructions
  2. Jailbreak - Bypassing safety guidelines and ethical constraints
  3. Role Confusion - Confusing the model about its role or identity
  4. Context Switching - Switching context, language, or format
  5. Indirect Injection - Injection through external data or documents
  6. Data Extraction - Extracting system prompts or training data

Defense Mechanisms

  1. Input Sanitization - Blocklist-based filtering
  2. Prompt Templating - Structured prompts with delimiters
  3. Output Filtering - Response content scanning
  4. Context Isolation - XML-based context separation
  5. Dual-LLM Verification - Guardian model validation

Technology Stack

  • Backend: Python 3.12, Flask
  • LLM: Ollama (llama3.2)
  • Database: SQLite
  • Frontend: Bootstrap 5, jQuery
  • Deployment: Docker Compose

Research Contributions

  • Comprehensive attack pattern library
  • Quantitative comparison of defense mechanisms
  • Reproducible testing methodology
  • Interactive demonstration tool
  • Open-source implementation
